## Пошаговая установка

### Что нужно заранее

- Компьютер с Windows, macOS или Linux
- [Node.js](https://nodejs.org/en/download) — скачайте LTS-версию (18 или выше)
- Аккаунт на [Supabase](https://supabase.com) — бесплатно, без карты

> 💡 **Проверить Node.js:** откройте терминал и введите `node --version`. Должно показать `v18.x.x` или выше.

---

### Шаг 1 — Скачайте проект

**Вариант A — через git:**
```bash
git clone https://github.com/SardorTheBest/smart-scrum-board.git
cd smart-scrum-board
```

**Вариант B — zip-архив:**
1. Нажмите зелёную кнопку **Code** → **Download ZIP**
2. Распакуйте архив
3. Откройте папку в терминале

---

### Шаг 2 — Создайте базу данных (Supabase)

#### 2.1 Зарегистрируйтесь на Supabase

1. Перейдите на [supabase.com](https://supabase.com) и нажмите **Start your project**
2. Войдите через GitHub или создайте аккаунт
3. Нажмите **New project**
4. Заполните:
   - **Name** — любое название, например `scrum-board`
   - **Database Password** — придумайте сложный пароль и сохраните его
   - **Region** — выберите ближайший к вам регион
5. Нажмите **Create new project** и подождите ~1 минуту

#### 2.2 Получите ключи доступа

1. В левом меню нажмите ⚙️ **Project Settings**
2. Выберите вкладку **Data API**
3. Скопируйте два значения:

```
Project URL:    https://xxxxxxxxxxxxxxxxxx.supabase.co
                           ↑ это ваш VITE_SUPABASE_URL

anon / public:  eyJhbGciOiJIUzI1NiIsInR5c...
                           ↑ это ваш VITE_SUPABASE_ANON_KEY
```

> ⚠️ Берите только `anon` ключ. Ключ `service_role` — только для серверного кода.

#### 2.3 Создайте таблицы в базе данных

1. В левом меню нажмите **SQL Editor**
2. Нажмите **New query**
3. Откройте файл `supabase/schema.sql` из скачанного проекта, скопируйте всё содержимое, вставьте в редактор и нажмите **Run**
4. Повторите для `supabase/add-comments.sql`
5. Повторите для `supabase/add-partners.sql`

После каждого файла должно появиться сообщение `Success. No rows returned`.

#### 2.4 Отключите подтверждение email

1. В левом меню нажмите **Authentication**
2. Перейдите в **Email**
3. Найдите переключатель **Confirm email** и **выключите** его

> Это важно! Без этого пользователи не смогут войти сразу после регистрации.

---

### Шаг 3 — Настройте переменные окружения

1. В папке проекта найдите файл `.env.example`
2. Создайте копию с именем `.env`:

```bash
# macOS / Linux
cp .env.example .env

# Windows (командная строка)
copy .env.example .env
```

3. Откройте `.env` в любом редакторе (Notepad, VS Code) и вставьте ваши ключи:

```env
VITE_SUPABASE_URL=https://xxxxxxxxxxxxxxxxxx.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5c...
```

> 🔒 Файл `.env` хранится только у вас — он не загружается на GitHub.

---

### Шаг 4 — Установите зависимости и запустите

```bash
# Установите пакеты (один раз, занимает ~1-2 минуты)
npm install

# Запустите приложение
npm run dev
```

Откройте браузер и перейдите по адресу: **http://localhost:5173**

🎉 Приложение запущено!

## Как пользоваться

### Регистрация и вход

1. На стартовой странице нажмите **Регистрация / Sign Up**
2. Введите email и пароль (минимум 6 символов)
3. Нажмите **Зарегистрироваться** — вы сразу попадёте в приложение

### Создание первой доски

1. В левом меню нажмите **+ Новая доска**
2. Введите название доски и параметры спринта (дата начала, конца, название)
3. Нажмите **Создать**

### Добавление задачи

1. Нажмите кнопку **+** в любой колонке доски
2. Заполните:
   - **Название** — коротко и ясно, что нужно сделать
   - **Описание** — подробности (необязательно)
   - **Категория** — Feature, Bug, Design и т.д.
   - **Приоритет** — Low, Medium, High, Critical
   - **Story Points** — оценка сложности (1, 2, 3, 5, 8, 13)
   - **Дедлайн** — срок выполнения
   - **Исполнитель** — кто делает задачу
3. Нажмите **Создать задачу**

### Перемещение задач

Просто **перетащите карточку** из одной колонки в другую мышью:
- 📥 **Sprint Backlog** — задача запланирована
- 🔄 **In Progress** — задача в работе
- 👀 **Review** — ждёт проверки
- ✅ **Done** — выполнена

### Обсуждение задачи

1. **Кликните на карточку** задачи — откроется боковая панель
2. Справа вы увидите чат с комментариями
3. Напишите сообщение и нажмите **Enter** или **Ctrl+Enter**
4. Комментарии видят все участники доски **в реальном времени**

### Дедлайн-уведомления

1. Нажмите на 🔔 **иконку колокольчика** в левом меню
2. Нажмите **Разрешить уведомления** — браузер спросит подтверждение
3. Теперь за 24 часа до дедлайна вы получите push-уведомление + звуковой сигнал
4. Рядом с колокольчиком можно включить/выключить звук

### Бэклог

Перейдите в раздел **Backlog** (левое меню):
- Нажмите **+** для добавления задачи в бэклог
- Стрелками ↑↓ меняйте приоритет задач
- Нажмите **Move to Sprint** чтобы взять задачу в текущий спринт

### Аналитика

В разделе **Analytics** вы увидите:
- **Burndown chart** — как сгорают story points за спринт
- **По колонкам** — сколько SP в каждой колонке
- **По категориям** — распределение задач
- **Velocity** — процент выполнения спринта

## Пригласить друзей в доску

### Способ 1 — Через кнопку Invite (внутри приложения)

1. Откройте нужную доску
2. Нажмите кнопку **Invite** (значок человека с плюсом) в верхней части доски
3. Введите **email** друга или коллеги
4. Нажмите **Отправить приглашение**
5. Друг получит уведомление и сможет зайти в вашу доску

> Друг должен сначала зарегистрироваться на вашей версии приложения.

### Способ 2 — Через Partners (список контактов)

1. В левом меню нажмите **Partners** (иконка людей)
2. Нажмите **Добавить партнёра**
3. Введите email человека
4. Можно добавить псевдоним — как вы его называете (например «Саша дизайнер»)
5. Теперь при назначении исполнителей на задачи этот человек будет в вашем списке

### Что видит приглашённый

После принятия приглашения коллега видит:
- Ту же доску с теми же задачами
- Комментарии обновляются у всех в реальном времени
- Он может перетаскивать задачи и добавлять свои

## Деплой своей версии

### Что значит "задеплоить"

После `npm run build` создаётся папка `dist/` — это готовый сайт из HTML, CSS и JavaScript файлов. Именно эту папку вы загружаете на хостинг и она становится вашим сайтом в интернете.

```
npm run build  →  папка dist/  →  загружаете на Vercel/Netlify  →  ваш сайт в интернете
```

### Vercel — рекомендуется (бесплатно)

1. Загрузите проект на GitHub (если ещё не сделали)
2. Зарегистрируйтесь на [vercel.com](https://vercel.com)
3. Нажмите **Add New Project** → выберите ваш репозиторий
4. Vercel автоматически определит Vite
5. Перейдите в **Environment Variables** и добавьте:
   ```
   VITE_SUPABASE_URL        = https://ваш-проект.supabase.co
   VITE_SUPABASE_ANON_KEY   = eyJhbGci...
   ```
6. Нажмите **Deploy** — через минуту сайт будет по адресу `ваш-проект.vercel.app`

При каждом `git push` в main Vercel автоматически обновит сайт.

### Netlify (альтернатива)

```bash
npm run build
npx netlify deploy --prod --dir dist
```

## Безопасность

- 🔒 Файл `.env` добавлен в `.gitignore` — ключи никогда не попадут в репозиторий
- 🛡 **Row Level Security (RLS)** на всех таблицах — каждый пользователь видит только свои данные
- 🔑 `anon`-ключ безопасен в браузере — это публичный ключ, защита строится через RLS
- ❌ `service_role` ключ не используется на клиенте
- 🔐 Пароли пользователей хранит Supabase Auth (bcrypt хэширование)

**Если случайно закоммитили ключ:**
Supabase → Settings → API → нажмите **Regenerate** рядом с anon key. Старый ключ сразу станет недействительным.
